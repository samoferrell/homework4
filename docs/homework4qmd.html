What is a side-effect function?

A side-effect function is a function that doesn’t change anything it just creates output.

First, we will write a function that finds the RMSE of a response vector and a respective predictions vector.

# I will be adding ... into the function to apply other arguments after the fact. 
getRMSE <- function(responses, predictions,...){
  RMSE <- sqrt(mean((responses - predictions)^2,...))
  return(RMSE)
}

This function will take in a url. If no title_keywords are specified, it returns the whole data set. If title_keywords are specified, it will search the title to find matches and filter based on those matches.

access <- function(url_id, title_keywords = NULL, starting_year = 0,...){
  id_info <- httr::GET(url_id)
  parsed <- fromJSON(rawToChar(id_info$content))
  headline_info <- tibble::as_tibble(parsed$articles)
  # if no title keywords are specified, return the whole data set.
  if (is.null(title_keywords)){
  return(headline_info)}
  else {
  
  # lower case the title keywords and create a 
  lower_case <- tolower(title_keywords)
  vector_keywords <- paste(lower_case, collapse = "|")

   filtered <- headline_info |>
    filter(grepl(vector_keywords, tolower(title)),
           as.numeric(substr(publishedAt,1,4)) >= starting_year)
  # if title keywords are specified, we will use str_detect to find look through and filter the titles.
  return(filtered)}
}
